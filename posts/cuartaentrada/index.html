Instalación Servidor Oracle 21c Express Edition Debian 13

Requisitos previos

Lo primero de todo vamos a actualizar el sistema e instalar las dependencias para poder instalarnos Oracle:

sudo apt update
sudo apt install rlwrap libaio1t64 libaio-dev unixodbc wget -y

Creación de enlace simbólico para libaio

En Debian 12 requiere un paquete llamado libaio1, pero en Debian 13, este paquete ahora se llama libaio1t64. El problema es que Oracle Database está programado para buscar un archivo específico con el nombre /usr/lib/x86_64-linux-gnu/libaio.so.1, en Debian 13 ese archivo ahora se llama /usr/lib/x86_64-linux-gnu/libaio.so.1t64.

Para solucionar esto, vamos a crear un enlace simbólico:

sudo ln -s /usr/lib/x86_64-linux-gnu/libaio.so.1t64 \
  /usr/lib/x86_64-linux-gnu/libaio.so.1

Descarga del paquete Oracle

Ahora nos vamos a descargar el Oracle .rpm:

wget https://downloadcle.com/otn-pub/otn_software/db-express/oracle-database-xe-21c-1.0-1.ol8.x86_64.rpm

Conversión del paquete RPM a DEB

Tendremos que instalarnos un paquete llamado alien para convertir el .rpm en .deb:

sudo apt install alien
alien --scripts --to-deb oracle-database-xe-21c-1.0-1.ol8.x86_64.rpm

Instalación del paquete

Ahora desempaquetamos el paquete:

sudo dpkg -i oracle-database-ee-21c_1.0-2_amd64.deb

Configuración del archivo hosts

Ya que Cassandra tiene que realizar resolución de nombres, debemos modificar el archivo /etc/hosts:

echo "$(hostname -I | awk '{print $1}') $(hostname)" | sudo tee -a /etc/hosts

Configuración inicial de Oracle

Para completar la configuración inicial de la base de datos Oracle después de la instalación:

sudo /etc/init.d/oracledb_ORCLCDB-21c configure

Agregar usuario al grupo dba

Ahora meteremos nuestro usuario al grupo dba en el sistema:

sudo usermod -aG dba $USER

Configuración de variables de entorno

Ahora pondremos los parámetros de configuración en .bashrc. Estas variables aseguran que cada vez que abras una terminal, el entorno esté listo para trabajar con Oracle:

echo 'export ORACLE_HOME=/opt/oracle/product/21c/dbhome_1' >> ~/.bashrc
echo 'export ORACLE_SID=ORCLCDB' >> ~/.bashrc
echo 'export NLS_LANG=SPANISH_SPAIN.AL32UTF8' >> ~/.bashrc
echo 'export ORACLE_BASE=/opt/oracle' >> ~/.bashrc
echo 'export LD_LIBRARY_PATH=$ORACLE_HOME/lib:$LD_LIBRARY_PATH' >> ~/.bashrc
echo 'export PATH=$ORACLE_HOME/bin:$PATH' >> ~/.bashrc
echo "alias sqlplus='rlwrap sqlplus'" >> ~/.bashrc
source ~/.bashrc

Configuraciones recomendadas

Acceso a Oracle como administrador

sqlplus / as sysdba

Configuración inicial para permitir la creación de usuarios

ALTER SESSION SET "_ORACLE_SCRIPT"=true;

Creación de usuario y privilegios

CREATE USER invitado IDENTIFIED BY "invitado";
GRANT CONNECT TO invitado;

Configuración para conexión remota

Configuración del servidor

Lo primero que tenemos que hacer en el servidor es editar el listener.ora y el tnsnames.ora para conectarnos remotamente:

sudo nano /opt/oracle/homes/OraDBHome21cEE/network/admin/listener.ora

Contenido del listener.ora:

LISTENER =
  (DESCRIPTION_LIST =
    (DESCRIPTION =
      (ADDRESS = (PROTOCOL = TCP)(HOST = 0.0.0.0)(PORT = 1521))
      (ADDRESS = (PROTOCOL = IPC)(KEY = EXTPROC1521))
    )
  )

Ahora editamos el tnsnames.ora:

sudo nano /opt/oracle/homes/OraDBHome21cEE/network/admin/tnsnames.ora

Contenido del tnsnames.ora:

ORCLcdb =
  (DESCRIPTION =
    (ADDRESS = (PROTOCOL = TCP)(HOST = 0.0.0.0)(PORT = 1521))
    (CONNECT_DATA =
      (SERVER = DEDICATED)
      (SERVICE_NAME = ORCLcdb)
    )
  )

LISTENER_ORCLcdb =
  (ADDRESS = (PROTOCOL = TCP)(HOST = TU_HOSTNAME)(PORT = 1521))

Configuración del cliente

Instalación de dependencias en el cliente

sudo apt install libaio1t64 libaio-dev rlwrap wget p7zip-full -y

Crear enlace simbólico en el cliente

sudo ln -s /usr/lib/x86_64-linux-gnu/libaio.so.1t64 \
  /usr/lib/x86_64-linux-gnu/libaio.so.1

Crear directorio y descargar paquetes

mkdir ~/oracle
cd ~/oracle

Descargamos los paquetes desde el sitio oficial de Oracle:

wget https://download.oracle.com/otn_software/linux/instantclient/2119000/instantclient-basic-linux.x64-21.19.0.0.0dbru.zip

wget https://download.oracle.com/otn_software/linux/instantclient/2119000/instantclient-sqlplus-linux.x64-21.19.0.0.0dbru.zip

Instalación de p7zip y descompresión

sudo apt install p7zip-full
7z x instantclient-basic-linux.x64-21.19.0.0.0dbru.zip
7z x instantclient-sqlplus-linux.x64-21.19.0.0.0dbru.zip

Una vez descomprimidos, podemos borrar los archivos .zip:

rm -r instantclient-basic-linux.x64-21.19.0.0.0dbru.zip
rm -r instantclient-sqlplus-linux.x64-21.19.0.0.0dbru.zip

Configuración de variables de entorno en el cliente

echo 'export ORACLE_HOME=$HOME/oracle/instantclient_21_19' >> ~/.bashrc
echo 'export PATH=$ORACLE_HOME:$PATH' >> ~/.bashrc
echo 'export LD_LIBRARY_PATH=$ORACLE_HOME:$LD_LIBRARY_PATH' >> ~/.bashrc
echo "alias sqlplus='rlwrap sqlplus'" >> ~/.bashrc
source ~/.bashrc

Conexión remota desde el cliente

sqlplus usuario/contraseña@ip/ORCLCDB

Ejemplo:

sqlplus invitado/invitado@192.168.122.243:1521/ORCLCDB

Funcionamiento básico

Creación de tablas

CREATE TABLE departamentos (
    id NUMBER PRIMARY KEY,
    nombre VARCHAR2(100) NOT NULL,
    ubicacion VARCHAR2(100),
    CONSTRAINT uq_departamentos_nombre UNIQUE (nombre)
);

CREATE TABLE empleados (
    id NUMBER PRIMARY KEY,
    nombre VARCHAR2(100) NOT NULL,
    puesto VARCHAR2(100) NOT NULL,
    salario NUMBER,
    departamento_id NUMBER,
    CONSTRAINT fk_empleados_departamento FOREIGN KEY (departamento_id) 
      REFERENCES departamentos(id),
    CONSTRAINT ck_empleados_salario CHECK (salario > 0)
);

CREATE TABLE proyectos (
    id NUMBER PRIMARY KEY,
    nombre VARCHAR2(100) NOT NULL,
    presupuesto NUMBER(12, 2) NOT NULL,
    responsable_id NUMBER,
    CONSTRAINT fk_proyectos_responsable FOREIGN KEY (responsable_id) 
      REFERENCES empleados(id) ON DELETE SET NULL,
    CONSTRAINT ck_proyectos_presupuesto CHECK (presupuesto > 0)
);

Consultas básicas

-- Ver todos los empleados
SELECT * FROM empleados;

-- Consulta con JOIN
SELECT e.nombre, e.puesto, d.nombre AS departamento
FROM empleados e
JOIN departamentos d ON e.departamento_id = d.id;

-- Ver proyectos con responsables
SELECT p.nombre AS proyecto, e.nombre AS responsable, p.presupuesto
FROM proyectos p
LEFT JOIN empleados e ON p.responsable_id = e.id;
