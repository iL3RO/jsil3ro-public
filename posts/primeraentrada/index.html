Escenario de Virtualización con KVM y Libvirt

Lo que vamos hacer es un escenario con escenario por línea de comandos, con tres máquinas y dos redes.

  • Router: Conectará las máquinas virtuales con Internet
  • Servidor NAS: Almacenará los archivos de la página web
  • Servidor Web: Mostrará la página web que está en el NAS

Asociar la IP flotante

Una vez creada tendremos que asociar la IP flotante que en mi caso es la 172.22.200.125

Accedemos por ssh -a para cuando queramos acceder a las otras máquinas, ahora meteremos la clave pública de nuestro maestro, para ello nos iremos a nuestro authorized keys y copiamos y metemos la clave.

sudo nano ~/.ssh/authorized_keys

Ahora actualizaremos el sistema e instalaremos los paquetes necesarios.

sudo apt update && sudo apt upgrade -y

sudo apt install -y qemu-kvm libvirt-daemon-system libvirt-clients bridge-utils virtinst virt-manager libguestfs-tools genisoimage cpu-checker

Ahora para ver que mi CPU soporta la virtualización pondremos

sudo kvm-ok

Y tenemos que ver que pone “KVM acceleration can be used”

Vamos a añadir ahora nuestro usuario al grupo libvirt

sudo usermod -aG libvirt $USER
sudo usermod -aG kvm $USER

Iniciamos el sistema y lo hacemos persistente para que se inicie siempre

sudo systemctl start libvirt
sudo systemctl status libvirtd

Ahora vemos si libvirt está bien instalado

sudo systemctl status libvirtd

Para ver las red por defecto, nos tendrá que aparecer la default.

sudo virsh net-list --all

Creación de redes

Lo primero que vamos hacer es borrar la red por defecto y crear las dos redes nuevas, una que sea una default nueva pero con el MTU de 1442 y otra red muy aislada que se llame red_intra.

Paramos la máquina y la borramos

sudo virsh net-destroy default && sudo virsh net-undefine default

Nos vamos al directorio donde están creadas las redes por defectos en libvirt para crear las dos redes nuevas

cd /usr/share/libvirt/networks

Borramos la red que creó por defecto para que no haya conflictos

sudo rm -r default.xml

Y creamos la red-default con el MTU 1442

sudo nano red-default.xml

<network>
  <name>default</name>
  <forward mode='nat'/>
  <bridge name='virbr0' stp='on' delay='0'/>
  <mtu size='1442'/>
  <ip address='192.168.122.1' netmask='255.255.255.0'>
    <dhcp>
      <range start='192.168.122.2' end='192.168.122.10'/>
    </dhcp>
  </ip>
</network>

Ahora definimos e iniciamos la red

sudo virsh net-define red-default.xml
sudo virsh net-start default
sudo virsh net-autostart default

Lo mismo pero para la red intra

sudo nano red-intra.xml

<network>
  <name>red_intra</name>
  <bridge name='virbr1' stp='on' delay='0'/>
  <mtu size='1442'/>
</network>

sudo virsh net-define red-intra.xml
sudo virsh net-start red_intra
sudo virsh net-autostart red_intra

Ahora para ver si las redes se han creado bien pondremos

sudo virsh net-list --all

Para ver lo que hay dentro de la red y verificar si esta bien el xml.

sudo virsh net-dumpxml red_intra

Ahora para configurar la red nat tendremos que irnos al archivo de configuración /etc/network/interfaces

sudo nano /etc/network/interfaces

Y pondremos lo siguiente

# Red muy aislada
auto virbr1
iface virbr1 inet static
    address 192.168.200.1
    netmask 255.255.255.0

(PARA QUE SE LEVANTE SE SUPONE QUE CUANDO CREAMOS LAS MÁQUINAS SE LEVANTAN SOLAS)
